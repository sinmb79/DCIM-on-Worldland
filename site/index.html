Why DCIP

AI coordination should be infrastructure, not a product silo.

The project documents frame DCIP as an open protocol layer: inference happens off-chain, proof happens on-chain, and ACL issuance emerges from participation instead of administrative discretion.

01

No single owner

Deployment is a setup step, not a governance role. The token owner renounces control after wiring the protocol.

02

Proof of Inference

Each result is hashed, stored on IPFS, signed by validators, and verified on Worldland before ACL is minted.

03

Humans and agents

AI nodes, validator nodes, human participants, and bounty solvers all plug into the same incentive surface.

Tokenomics

Bitcoin-style issuance, calibrated for 10-second blocks.

ACL starts at 84 tokens per block, halves every 12,614,400 blocks, and routes 100% of each reward to participants.

Participate

Use the protocol now, not just read about it.

The MVP site connects a Worldland wallet, sends a query to a DCIP node, and surfaces the resulting answer, transaction hash, and IPFS CID.

Live Query

Submit a query to the collective intelligence network.

Node Operator

Run inference, collect signatures, submit proof.

Configure the root .env, start the TypeScript node server, and register your endpoint with DCIPRegistry.registerNode using the helper functions in node-server/src/chain.ts.

Validator

Stay online and sign proofs.

Validator nodes use the same API server, expose /sign, and earn from the 40% validator share when their signatures make it on-chain.

Bounty Poster

Escrow ACL and reward the best solution.

Post an IPFS-backed problem statement, accept solution CIDs, and release ACL automatically to the winning solver with Bounty.approveSolution.

Roadmap

            
v1 ships the protocol spine, not every whitepaper ambition.

          

          

            This implementation focuses on contract deployment, node-server operation, and a usable query surface. Anti-spam economics, fee burns, and zkML remain future phases.

            Core contracts
